# SpeakFlow 语音输入法

SpeakFlow 是一个帮助用户提升文本输入效率的浏览器端语音输入法 MVP。项目面向课堂笔记、会议纪要、聊天短句、学术草稿等场景，重点平衡准确度、易用性、响应速度和成本。

## 一、产品定位

### 目标用户

- 需要快速记录课堂、会议、灵感的学生与办公用户
- 打字速度较慢、希望用语音完成初稿输入的用户
- 需要低成本、无需复杂安装的轻量级语音输入工具用户

### 核心痛点

1. 传统键盘输入速度有限，长文本记录效率低。
2. 纯语音转文字工具通常缺少编辑辅助能力。
3. 很多语音识别方案依赖付费云 API，成本较高。
4. 输入法类产品需要响应快、交互简单，不能打断用户表达。

## 二、核心功能

| 模块 | 功能 |
|---|---|
| 语音识别 | 支持中文普通话和英文语音输入 |
| 实时反馈 | 展示 interim 临时识别结果和 final 最终文本 |
| 自动标点 | 支持“逗号、句号、问号、感叹号”等口述标点转换 |
| 语音命令 | 支持换行、删除上一句、复制全文、清空文本 |
| 文本编辑 | 识别后可继续手动修改 |
| 效率统计 | 显示字符数、句子数、识别片段数、响应延迟 |
| 成本控制 | 使用浏览器 Web Speech API，MVP 阶段无需自建后端和付费 API |
| 导出能力 | 支持复制全文和下载 TXT |

## 三、创新点

1. **语音输入 + 语音编辑命令结合**：不仅转文字，还能通过“删除上一句”“复制全文”等命令完成轻量编辑。
2. **场景化输入模式**：保留课堂/会议笔记、聊天短句、学术写作草稿模式入口，便于后续扩展不同文本后处理策略。
3. **低成本架构**：MVP 使用浏览器端能力，无需服务器，部署简单，API 成本为 0。
4. **可解释的效率反馈**：提供字符数、句子数、识别片段、响应延迟，让用户直观看到输入效率。

## 四、技术架构

```text
用户语音
  ↓
Browser Web Speech API
  ↓
实时识别结果 interim / final
  ↓
文本后处理模块
  ├─ 自动标点
  ├─ 语音命令识别
  └─ 场景化格式处理
  ↓
编辑器 textarea
  ↓
复制 / 下载 / 统计 / 日志
```

## 五、技术选型

| 类型 | 方案 | 原因 |
|---|---|---|
| 前端 | HTML + CSS + JavaScript | 简单稳定，便于评委直接运行 |
| 语音识别 | Web Speech API | 无需后端，低成本，响应快 |
| 数据存储 | 当前版本本地内存 | MVP 聚焦输入效率，不收集隐私数据 |
| 部署 | GitHub Pages / Gitee Pages | 静态网页即可部署 |

## 六、运行方式

### 方式 1：直接打开

双击 `index.html`，建议使用 Chrome 或 Edge 浏览器。

### 方式 2：本地静态服务

```bash
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

## 七、浏览器兼容性说明

当前 MVP 使用浏览器 Web Speech API。建议在 Chrome 或 Edge 中运行。若浏览器不支持，页面会显示“不可用”状态。

## 八、如何使用

1. 选择识别语言：中文普通话或 English。
2. 选择应用场景：课堂/会议笔记、聊天短句或学术写作草稿。
3. 点击“开始语音输入”。
4. 说出文本内容，例如：
   - “今天的会议主要讨论模型优化方案句号”
   - “换行下一步需要完成 demo 视频句号”
5. 使用语音命令：
   - “删除上一句”
   - “复制全文”
   - “清空文本”
6. 点击“复制”或“下载 TXT”导出结果。

## 九、准确度、易用性、响应速度、成本平衡

| 指标 | 设计策略 |
|---|---|
| 准确度 | 使用浏览器成熟语音识别能力；保留手动编辑区；支持标点后处理 |
| 易用性 | 单页面操作；按钮清晰；语音命令可见；无需登录 |
| 响应速度 | 前端直接调用浏览器识别接口；实时显示 interim 结果 |
| 成本 | 不使用付费云 API；无后端服务器；静态部署 |

## 十一、后续可扩展方向

- 接入 Whisper / FunASR 等本地或服务端模型，提升识别稳定性。
- 增加用户词库，例如课程名、人名、专业术语。
- 增加 LLM 文本润色：语音初稿 → 自动整理成笔记。
- 支持快捷键唤醒和悬浮输入框，更接近真实输入法体验。
- 支持离线识别方案，进一步保护隐私。

## 十二、第三方依赖与原创说明

当前版本无 npm 依赖，未引入第三方前端框架。核心原创功能包括：页面交互、语音命令处理、自动标点处理、文本统计、日志反馈、导出功能和整体产品设计。
