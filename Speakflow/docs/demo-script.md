# Demo 视频讲解脚本

## 1. 开场（约 15 秒）

大家好，这是我的作品 SpeakFlow 语音输入法。它的目标是帮助用户通过语音快速输入、编辑和导出文本，适用于课堂笔记、会议纪要、聊天短句和学术写作等场景。

## 2. 产品价值（约 20 秒）

传统键盘输入在长文本记录时效率较低。普通语音转文字工具通常只做转写，缺少编辑命令和效率反馈。SpeakFlow 在准确度、易用性、响应速度和成本之间找到了平衡——基于浏览器内置 API，零成本、零安装。

## 3. 功能展示（约 90 秒）

### 3.1 基础识别

打开页面（Chrome 浏览器），可以看到页面上方的状态卡片显示「可用」徽章。

我选择中文普通话，课堂笔记模式，开启自动标点和语音命令。

点击「开始语音输入」，浏览器弹出麦克风授权，点击允许。

现在我说话，大家可以注意三个区域：
- 底部临时结果栏实时显示 interim 识别内容（脉冲动画指示）
- 确认后的 final 文本进入编辑区
- 右侧统计卡片实时更新字符数、句子数和识别片段数

示例语音：“今天我们讨论语音输入法的核心功能句号第一点是提高文本输入效率分号第二点是降低使用成本句号”

可以看到文本被实时识别到编辑区，且「句号」「分号」自动转换为标点符号。

### 3.2 多语言支持

现在切换到 English，说："This is a voice input method demo period"

可以看到英文也能正常识别，标点处理同样生效。

### 3.3 语音命令展示

接下来展示语音命令功能。

已有文本的情况下，说「换行」——编辑区插入换行。

说「删除上一句」——最后一句文本被删除。

说「复制全文」——文本复制到剪贴板，顶部出现「已复制到剪贴板」提示，复制按钮变绿。

说「清空文本」——编辑区完全清空，所有统计归零。

### 3.4 导出功能

手动输入一段文本，然后点击「下载 TXT」按钮，浏览器会下载一个带日期时间的 txt 文件。

### 3.5 快捷键

在录音过程中按键盘的 `Esc` 键，录音立即停止，操作日志记录快捷键停止事件。

## 4. 错误处理展示（约 20 秒）

如果我用不支持 Web Speech API 的浏览器（如 Firefox）打开，页面会显示「不可用」徽章，开始按钮被禁用。

如果用户拒绝麦克风权限，日志中会显示「麦克风权限被拒绝」。

在录音中静默不说话，页面保持稳定运行不崩溃。

## 5. 技术说明（约 20 秒）

本项目使用纯 HTML、CSS 和 JavaScript 实现，零第三方依赖。语音识别基于浏览器内置的 Web Speech API，数据仅在浏览器本地处理，无需后端服务器，成本为零。

代码采用 IIFE 模块化组织，分为元素绑定、兼容检测、识别引擎、文本处理、状态管理、指标统计、剪贴板操作、文件导出、日志记录九个层次。

## 6. 创新点总结（约 15 秒）

- 语音识别 + 语音编辑命令的结合，不只是转写工具
- 7 种口述标点自动转换，减少手动编辑
- 场景化文本后处理策略
- 四维实时效率统计（字符数、句子数、识别片段、响应延迟）
- 零依赖架构，评委可直接双击 index.html 运行
