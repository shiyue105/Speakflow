# 用户需求分析

## 1. 用户画像

### 学生用户

- 场景：课堂笔记、小组讨论、论文草稿、作业记录。
- 需求：快速把口头想法变成文本，后续再整理。
- 痛点：打字慢、听课时难以同时理解和记录。

### 办公用户

- 场景：会议纪要、任务记录、日报周报草稿。
- 需求：快速形成可复制、可导出的文字。
- 痛点：会议内容多，手动记录容易遗漏。

### 轻量创作者

- 场景：短文、脚本、灵感记录。
- 需求：降低从想法到文字的阻力。
- 痛点：键盘输入打断表达节奏。

## 3. 产品设计原则

1. 尽量减少用户操作步骤。
2. 识别结果必须可编辑，避免因识别错误导致不可用。
3. MVP 阶段优先保证低成本和可演示。
4. 通过语音命令增强“输入法”属性，而不是只做普通转写工具。
